### 3. Translation Language Verification (Critical Fix)

**File:** `core_v2/orchestrator.py`

**Problem:** LLM sometimes returned untranslated chunks (e.g., Vietnamese text when target was English), causing 65% of Chi Pheo output to remain in Vietnamese.

**Fix — 3 additions:**

1. **`_detect_language(text)` static method** — Character-based language detection:
   - Vietnamese: diacritical marks (ắ, ẳ, ê, ơ, ư, đ, etc.)
   - Chinese/Japanese: CJK unicode ranges
   - English: Latin without diacriticals
   - Min 20 chars threshold

2. **Post-translation verification in `_translate_chunk()`**:
   ```python
   detected = self._detect_language(translated)
   if detected != target_lang:
       # Retry with explicit "ONLY output in {target_lang}" prompt
   ```

3. **Quality summary in `_translate_chunks()`**:
   ```
   Translation quality: 18/20 chunks in target language 'en'
   Chunks still in source language: [3, 7]
   ```

## Architecture

Frontend (Next.js 16, port 3000)

        └── Backend (FastAPI, port 8000)
              ├── /api/v2/jobs — V2 job CRUD
              ├── /api/v2/publish/* — Translation pipeline
              └── core_v2/orchestrator.py
                    ├── _translate_chunk() — with language verification
                    ├── _detect_language() — vi/en/zh/ja detection
                    └── _assemble() — join translated chunks
