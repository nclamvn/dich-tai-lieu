## P0 CRITICAL FINDINGS (8 items)

These must be fixed **before any production or multi-user deployment**.

### SEC-01: Arbitrary File Read via `/api/analyze`
- **File:** `api/routes/uploads.py:86-88`
- **Issue:** Accepts client-supplied `file_path` without `validate_project_path()`. Attacker can read any file on the server (e.g., `/etc/passwd`, `.env`).
- **Fix:** Call `validate_project_path(request.file_path)` before use.

### SEC-02: Authentication Disabled by Default
- **File:** `config/settings.py:54-58`
- **Issue:** `session_auth_enabled = False`, `security_mode = "development"`. All endpoints are publicly accessible.
- **Fix:** Enable auth by default or add startup warning. Gate sensitive routes.

### SEC-03: Password Reset Token Leaked in API Response
- **File:** `api/auth_router.py:134-140`
- **Issue:** `_dev_token` returned in response body. Comment says "Remove in production" but no env guard exists.
- **Fix:** Remove `_dev_token` or gate behind `settings.security_mode == "development"`.

### SEC-04: WebSocket Unauthenticated
- **File:** `api/main.py:409-454`
- **Issue:** `/ws` endpoint accepts any connection without auth. All job events broadcast to all clients.
- **Fix:** Check session token in WebSocket handshake.

### DATA-01: No Multi-tenancy on Jobs
- **File:** `api/job_repository.py:43-68`, `core/job_queue.py:206-257`
- **Issue:** `aps_jobs` and `jobs` tables have no `user_id` column. All users see all jobs.
- **Fix:** Add `user_id TEXT` column, enforce in all queries.

### DATA-02: Screenplay User ID is Stub
- **File:** `api/routes/screenplay.py:45-47`
- **Issue:** `get_user_id()` always returns `"default_user"`. Despite per-user isolation checks, all users share one identity.
- **Fix:** Wire to actual auth session.

### DATA-03: V2 Bulk Delete Ignores Running Jobs
- **File:** `api/aps_v2_router.py:234-249`
- **Issue:** `DELETE /api/v2/jobs` deletes DB records regardless of status. Running jobs become orphaned background tasks.
- **Fix:** Check status before delete, cancel task first.

### PERF-01: No Global Concurrency Limit
- **File:** `core_v2/orchestrator.py:291`
- **Issue:** Per-job semaphore only. 50 simultaneous users = 250 concurrent LLM API calls. No backpressure.
- **Fix:** Add global `MAX_CONCURRENT_JOBS` semaphore in `APSV2Service`.
